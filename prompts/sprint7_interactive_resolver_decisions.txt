SPRINT-7 OBJECTIVE

Provide a UI and backend mechanism allowing users to override column interpretations when automatic detection may be incorrect.

Examples:

Likert scale incorrectly detected as text
Boolean detected column should be treated as nominal
Column should be ignored during analysis

---

OVERRIDE MODEL

Create a new model:

app/models/overrides.py

Define:

ColumnOverride

Fields:

dataset_id: str
column_name: str
override_type: str

Allowed override_type values:

LIKERT
BOOLEAN
NUMERIC
TEXT
IGNORE

Overrides should be stored in a simple JSON file:

storage/overrides/{dataset_id}_overrides.json

---

UI PAGE

Add a new page:

GET /datasets/{dataset_id}/overrides

Display:

• column names
• inferred type (from dataset_schema.json)
• dropdown selector with override options

Override options:

Default (no override)
Likert
Boolean
Numeric
Text
Ignore column

Form submission:

POST /datasets/{dataset_id}/overrides

Save overrides to JSON file.

---

PREVIEW UPDATE

Modify dataset preview page:

GET /datasets/{dataset_id}/preview

Display a note when overrides exist:

"Overrides applied: X columns"

Ignored columns should not appear in preview.

overrides.json

}
